  was causing the issue and what I fixed:

  The Problem

  The error "type double is not a subtype of type String"
  was occurring because:

  1. The database was returning price and duration fields
  as numeric types (likely double or int)
  2. The code was trying to directly cast them to String
  using ?.toString() on the nullable value
  3. When the field wasn't null but was a double, the cast
   failed

  The Solution

  I fixed this in two methods in
  /Users/adityapaswan/soup/projects/janjeevan/vidyaras_app
  /lib/src/features/my_learning/4_data/datasources/my_lear
  ning_remote_datasource.dart:

  Lines 115-124 (getEnrolledCourses method)
  Lines 214-223 (getCourseDetails method)

  The fix handles both cases:
  - If the value is already a String, it casts it directly
  - If the value is a numeric type (like double), it
  converts it to String using .toString()
  - If the value is null, it returns null

  price: courseData['price'] != null
      ? (courseData['price'] is String
          ? courseData['price'] as String
          : courseData['price'].toString())
      : null,
  duration: courseData['duration'] != null
      ? (courseData['duration'] is String
          ? courseData['duration'] as String
          : courseData['duration'].toString())
      : null,

  This approach is type-safe and handles all possible
  scenarios without throwing runtime errors. The learning
  screen should now load without the type casting error.
